Describe your process. At a high level, how did you go about creating this project?

     My inspiration for this project is that I have been doing research on interacting particle systems in another class and wanted to make a very accessible and interactive version of the simulations we were running. I used an agentic process with github copilot.


What AI tools and strategies did you use? What models did you choose, and were you using them in a browser chat, in something like Cursor, in the command line, etc? Did you use an agentic process like in HW8, or something else?

    I used github copilot primarily, but had to haphazrdly migrate to cursor to do the review stage of the process because I hit my usage limits. I used an agentic approach. I wrote a detailed SPEC.md and threw the LLM at it. Something slightly different about this approach was that I also provided it with the file "python reference code" that contained python implementations for the simulations that I wanted to make interactive and accessible from a website.


Why did you make the choices above?

    Github copilot has given me the best results so far, and the scope of the project was large enough that I wouldn't have been able to feasibly use my preferred method of AI-assisted programming of writing 1 function at a time. 


What changed from your pre-113 approach? Compare your approach to how you might have approached the project before 15-113. What has changed, if anything? Do you feel more or less well-equipped to create projects like this within a reasonable time frame?

    Pre 113 I had no experience dealing with AI beyond the google search overview, so everything has changed. Initially I started using browser chat AIs, but eventually discovered how much more effective copilot could be. I'm definitely more equipped to make ambitious projects like this, but I think I have a lot more to go before I will be equal with my peers who had already been familiar with AI. I still am not very used to the process and don't think I have the instincts needed to improvise when working with AI. When something goes awry I run to Mike's OH, which is honestly good because that resource is severley underutilized, but the point is I feel like the moment the AI is giving me unexpected behavior I am out of my depth and don't know what to do without consulting outside help.  


What might you do with this project with more time?

    The project is honestly not in a very good state relative to my ambitions. It is not deployed as web app, it has significantly slower performance than expected, and it has some bugs with the UI. I honestly think that once my usage resets, I might try to either start over from an updated SPEC sheet, or even use the more time consuming approach of building 1 function at a time.
